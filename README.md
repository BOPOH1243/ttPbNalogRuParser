# ttPbNalogRuParser
парсер ИП pb.nalog.ru
## реализовано:
- парсер
    - Сбор руководителей
    - Сбор организаций этих руководителей
    - пришлось поменять формат, ибо никто не запрещает одному человеку быть директором нескольких организаций
- FastAPI с одной ручкой /search
- управление брокером
- docker-compose

# Запуск
- ```sudo docker-compose up -d``` чтобы поднять контейнеры ~~либо без `sudo` если пользователю права в докер проброшены~~
- Fastapi разворачивается на 8000 порту ```https://0.0.0.0:8000/docs``` там по классике видна эта несчастная ручка
- После сбора данные отдаются в http ответом, а также кидаются в топик


##P.S.
Требование 30 секунд на запрос и отдавать данные в http ответе поставило меня в тупик, ибо прокси у меня нет, а быстрые запросы без интервала приводят к активации защиты (по моему там капча выскакивает). В общем я:
1. **Уменьшил объём сбора до первых 10 человек**
2. также через асинхронный контекстный менеджер ограничил скорость запросов (вернее задал интервал).
долгий сбор с интервалом приводит к тому что браузер разрывает соединение по таймауту.
